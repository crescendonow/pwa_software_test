# แผนปรับปรุง software_test (PWA GIS UAT) — 15 ก.ค. 2569

## เป้าหมาย

1. เปลี่ยนส่วนเลือก Sessions จาก sidebar ด้านซ้ายเป็น navbar แนวนอนด้านบน เพื่อคืนพื้นที่แนวนอนให้ตารางและช่องหมายเหตุ
2. เพิ่มหัวข้อทดสอบสำหรับ layer `additional_requirement` เพื่อให้ผู้ทดสอบกรอกความต้องการเพิ่มเติมผ่านช่องหมายเหตุได้โดยตรง

## สิ่งที่ตรวจพบ

- หน้า `web/pwagis_uat.html` แบ่ง workbench เป็น sidebar Sessions กว้าง 330px และตารางผลทดสอบด้านขวา
- frontend มีชื่อภาษาไทยของ `additional_requirement` และฐานข้อมูลมี layer นี้ใน `ref_layer_name` แล้ว
- สาเหตุที่ layer ไม่ปรากฏในหัวข้อทดสอบคือยังไม่มี row ใน `ut_logs.test_cases`; session จะสร้างผลเฉพาะ test case ที่ active ของเวอร์ชันนั้น

## แผนดำเนินการ

### A. Sessions navbar

- ย้ายหัวข้อ Sessions, จำนวน session, ตัวกรองสาขา และรายการ session ไปไว้ใน navbar เต็มความกว้างเหนือ workbench
- แสดงรายการ session เป็นการ์ดแนวนอนที่เลื่อนได้ เพื่อรองรับหลาย session และหน้าจอขนาดเล็ก
- คงสถานะ active, รายละเอียดเวอร์ชัน/วันที่/ผู้ทดสอบ/สาขา, การเลือก session และสิทธิ์ลบ session เดิม
- ขยายตารางผลทดสอบให้ใช้ความกว้างเต็มพื้นที่ เพื่อเพิ่มพื้นที่ให้ textarea หมายเหตุ

### B. Layer ความต้องการเพิ่มเติม

- เพิ่ม migration แบบ idempotent สำหรับ reference ที่ใช้กับหัวข้อ “ความต้องการเพิ่มเติม”
- เพิ่ม test case active ของ layer `additional_requirement` ให้ทั้งเวอร์ชัน `22 เมษายน 2569` และ `5 พฤษภาคม 2569`
- ใช้หัวข้อที่สื่อให้ผู้ทดสอบกรอกรายละเอียดในช่องหมายเหตุ และใช้ sort order ต่อท้ายรายการของแต่ละเวอร์ชัน
- migration ต้องสร้างผลรายการใหม่ให้ session เดิมของสองเวอร์ชันด้วย เพื่อไม่ให้ feature ใช้ได้เฉพาะ session ที่สร้างหลัง deploy

### C. การตรวจสอบ

- ตรวจ syntax JavaScript และรัน `go test ./...`
- ตรวจ SQL เชิงโครงสร้างว่า migration มี reference, test case และ backfill ครบ รวมทั้งทำงานซ้ำได้
- review diff เทียบกับ plan นี้ทั้งด้านมาตรฐานของ repository และความครบถ้วนตาม requirement

## Acceptance criteria

- Sessions แสดงเป็น navbar แนวนอนเหนือพื้นที่กรอกผล และตารางไม่มี sidebar เบียดด้านซ้าย
- เลือก กรอง และลบ session ได้เหมือนเดิม
- ช่องหมายเหตุมีพื้นที่แนวนอนมากขึ้นทั้ง desktop และ mobile (ตารางยังเลื่อนแนวนอนได้)
- เมื่อเปิด session ของทั้งสองเวอร์ชัน จะเห็น layer “ความต้องการเพิ่มเติม” และสามารถบันทึกข้อความในหมายเหตุได้
- session เดิมได้รับรายการ “ความต้องการเพิ่มเติม” โดยไม่ต้องสร้าง session ใหม่
- test suite และ syntax checks ผ่าน
